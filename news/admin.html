Student Records

According to our problem, the admin should be able to allot students to different projects. Instead of creating student database by default, we added a Student Records table so that the admin can add, edit or delete a student record and then allot that student to a specific project. The admin can create,edit or delete new projects and allot the students added to the project form. The admin can also change the allotment of the students by editing the project information.
